---
title: "The Sprint"
description: "Think → Plan → Build → Review → Test → Ship → Reflect"
---

jstack is a process, not a collection of tools. The skills run in the order a sprint runs:

```
Think → Plan → Build → Review → Test → Ship → Reflect
```

Each skill feeds into the next. `/office-hours` writes a design doc that `/plan-ceo-review` reads. `/plan-eng-review` writes a test plan that `/qa` picks up. `/review` catches bugs that `/ship` verifies are fixed. Nothing falls through the cracks because every step knows what came before it.

## Phases

<Steps>
  <Step title="Think">
    Start with `/office-hours` to reframe what you're building. Add `/design-consultation` to create a visual identity from scratch.
  </Step>
  <Step title="Plan">
    Run `/plan-ceo-review` to find the 10-star product, `/plan-eng-review` to lock architecture, and `/plan-design-review` to catch missing design states. Or use `/autoplan` to run all three automatically.
  </Step>
  <Step title="Build">
    Implement your feature. Use `/browse` for browser interaction and `/investigate` for systematic debugging.
  </Step>
  <Step title="Review">
    `/review` finds bugs that pass CI but break in production. `/design-review` audits the live UI. `/codex` provides a second opinion from a different AI. `/cso` runs a security audit.
  </Step>
  <Step title="Test">
    `/qa` opens a real browser, finds bugs, fixes them, and re-verifies. `/benchmark` detects performance regressions. `/canary` monitors post-deploy.
  </Step>
  <Step title="Ship">
    `/ship` runs tests, audits coverage, pushes, and opens the PR. `/land-and-deploy` merges and verifies in production. `/document-release` updates all docs.
  </Step>
  <Step title="Reflect">
    `/retro` generates a team-aware weekly retro with per-person breakdowns, shipping streaks, and test health trends.
  </Step>
</Steps>

## Smart review routing

Like a well-run startup: the CEO doesn't review infra bug fixes, design review isn't needed for backend changes. jstack tracks what reviews have run, figures out what's appropriate, and does the smart thing. The Review Readiness Dashboard tells you where you stand before you ship.
