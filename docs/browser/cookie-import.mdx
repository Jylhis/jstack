---
title: "Cookie Import"
description: "Import cookies from your real browser for authenticated testing"
---

jstack can import cookies from your real Chromium-based browsers for authenticated testing. Two methods:

## From a real browser

```bash
$B cookie-import-browser
```

Auto-detects installed browsers (Comet, Chrome, Arc, Brave, Edge), decrypts cookies via macOS Keychain, and loads them into the Playwright session.

Or use the `/setup-browser-cookies` skill for an interactive picker UI.

## From a file

```bash
$B cookie-import <path>
```

Import cookies from a JSON file.

## Security model

- **Keychain access requires user approval**: First import per browser triggers a macOS dialog
- **Decryption in-process**: Cookie values are decrypted in memory, never written to disk in plaintext
- **Read-only**: jstack copies the cookie DB to a temp file and opens it read-only
- **No values in logs**: Console and network logs never contain cookie values
- **Per-session caching**: Keychain password + AES key cached in memory, cleared on server shutdown

## Supported platforms

Currently macOS only. Linux (GNOME Keyring/kwallet) and Windows (DPAPI) are architecturally possible but not yet implemented.
