## The problem

SKILL.md files tell Claude how to use browse commands. If the docs list a flag that doesn't exist, or miss a command that was added, the agent hits errors. Hand-maintained docs always drift from code.

## The solution

```
SKILL.md.tmpl          (human-written prose + placeholders)
       ↓
gen-skill-docs.ts      (reads source code metadata)
       ↓
SKILL.md               (committed, auto-generated sections)
```

Templates contain workflows, tips, and examples. Placeholders are filled from source code at build time:

| Placeholder | Source | What it generates |
|-------------|--------|-------------------|
| `{{COMMAND_REFERENCE}}` | `commands.ts` | Categorized command table |
| `{{SNAPSHOT_FLAGS}}` | `snapshot.ts` | Flag reference with examples |
| `{{PREAMBLE}}` | `gen-skill-docs.ts` | Startup block: update check, session tracking, contributor mode |
| `{{BROWSE_SETUP}}` | `gen-skill-docs.ts` | Binary discovery + setup instructions |
| `{{BASE_BRANCH_DETECT}}` | `gen-skill-docs.ts` | Dynamic base branch detection |
| `{{QA_METHODOLOGY}}` | `gen-skill-docs.ts` | Shared QA methodology |
| `{{DESIGN_METHODOLOGY}}` | `gen-skill-docs.ts` | Shared design audit methodology |
| `{{REVIEW_DASHBOARD}}` | `gen-skill-docs.ts` | Review Readiness Dashboard |
| `{{TEST_BOOTSTRAP}}` | `gen-skill-docs.ts` | Test framework detection and setup |

If a command exists in code, it appears in docs. If it doesn't exist, it can't appear.

## Why committed, not generated at runtime?

1. **Claude reads SKILL.md at load time**: No build step when a user invokes `/browse`. The file must already exist.
2. **CI validates freshness**: `gen:skill-docs --dry-run` + `git diff --exit-code` catches stale docs before merge.
3. **Git blame works**: See when a command was added and in which commit.

## The preamble

Every skill starts with a `{{PREAMBLE}}` block that handles:

1. **Update check** — reports if an upgrade is available
2. **Session tracking** — counts active sessions, enters "ELI16 mode" at 3+ concurrent sessions
3. **Contributor mode** — files field reports when jstack misbehaves
4. **AskUserQuestion format** — consistent across all skills
5. **Search Before Building** — three layers of knowledge (tried-and-true, new-and-popular, first-principles)

## Workflow

```bash
# 1. Edit the template
vim SKILL.md.tmpl

# 2. Regenerate
bun run gen:skill-docs

# 3. Check health
bun run skill:check

# Or use watch mode
bun run dev:skill
```
