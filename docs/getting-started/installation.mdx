## Requirements

- [Claude Code](https://docs.anthropic.com/en/docs/claude-code)
- [Git](https://git-scm.com/)
- [Bun](https://bun.sh/) v1.0+
- [Node.js](https://nodejs.org/) (Windows only)

## Step 1: Install on your machine

Open Claude Code and paste this. Claude does the rest.

```
Install jstack: run git clone --single-branch --depth 1
https://github.com/Jylhis/jstack.git ~/.claude/skills/jstack
&& cd ~/.claude/skills/jstack && ./setup

Then add a "jstack" section to CLAUDE.md that says to use the /browse
skill from jstack for all web browsing, never use
mcp__claude-in-chrome__* tools, and lists the available skills.
```

## Step 2: Add to your repo (optional)

So teammates get jstack automatically:

```
Add jstack to this project: run cp -Rf ~/.claude/skills/jstack
.claude/skills/jstack && rm -rf .claude/skills/jstack/.git
&& cd .claude/skills/jstack && ./setup
```

Real files get committed to your repo (not a submodule), so `git clone` just works. Everything lives inside `.claude/`. Nothing touches your PATH or runs in the background.

<Tip>
**Contributing or need full history?** The commands above use `--depth 1` for a fast install. If you plan to contribute or need full git history, do a full clone instead:

```bash
git clone https://github.com/Jylhis/jstack.git ~/.claude/skills/jstack
```
</Tip>

## Codex, Gemini CLI, or Cursor

jstack works on any agent that supports the [SKILL.md standard](https://github.com/anthropics/claude-code). Skills live in `.agents/skills/` and are discovered automatically.

**Install to one repo:**

```bash
git clone --single-branch --depth 1 https://github.com/Jylhis/jstack.git .agents/skills/jstack
cd .agents/skills/jstack && ./setup --host codex
```

**Install once for your user account:**

```bash
git clone --single-branch --depth 1 https://github.com/Jylhis/jstack.git ~/jstack
cd ~/jstack && ./setup --host codex
```

**Auto-detect installed agents:**

```bash
git clone --single-branch --depth 1 https://github.com/Jylhis/jstack.git ~/jstack
cd ~/jstack && ./setup --host auto
```

All 28 skills work across all supported agents. Hook-based safety skills (careful, freeze, guard) use inline safety advisory prose on non-Claude hosts.

## Troubleshooting

<AccordionGroup>
  <Accordion title="Skill not showing up">
    ```bash
    cd ~/.claude/skills/jstack && ./setup
    ```
  </Accordion>

  <Accordion title="/browse fails">
    ```bash
    cd ~/.claude/skills/jstack && bun install && bun run build
    ```
  </Accordion>

  <Accordion title="Stale install">
    Run `/jstack-upgrade` — or set `auto_upgrade: true` in `~/.jstack/config.yaml`
  </Accordion>

  <Accordion title="Want shorter commands">
    ```bash
    cd ~/.claude/skills/jstack && ./setup --no-prefix
    ```
    Switches from `/jstack-qa` to `/qa`. Your choice is remembered for future upgrades.
  </Accordion>

  <Accordion title="Want namespaced commands">
    ```bash
    cd ~/.claude/skills/jstack && ./setup --prefix
    ```
    Switches from `/qa` to `/jstack-qa`. Useful if you run other skill packs alongside jstack.
  </Accordion>

  <Accordion title="Codex says 'Skipped loading skill(s) due to invalid SKILL.md'">
    Your Codex skill descriptions are stale. Fix:
    ```bash
    cd ~/.codex/skills/jstack && git pull && ./setup --host codex
    ```
    For repo-local installs:
    ```bash
    cd "$(readlink -f .agents/skills/jstack)" && git pull && ./setup --host codex
    ```
  </Accordion>

  <Accordion title="Windows users">
    jstack works on Windows 11 via Git Bash or WSL. Node.js is required in addition to Bun — Bun has a known bug with Playwright's pipe transport on Windows. The browse server automatically falls back to Node.js. Make sure both `bun` and `node` are on your PATH.
  </Accordion>

  <Accordion title="Claude says it can't see the skills">
    Make sure your project's `CLAUDE.md` has a jstack section:
    ```
    ## jstack
    Use /browse from jstack for all web browsing.
    Never use mcp__claude-in-chrome__* tools.
    Available skills: /office-hours, /plan-ceo-review, /plan-eng-review,
    /plan-design-review, /design-consultation, /review, /ship,
    /land-and-deploy, /canary, /benchmark, /browse, /qa, /qa-only,
    /design-review, /setup-browser-cookies, /setup-deploy, /retro,
    /investigate, /document-release, /codex, /cso, /autoplan,
    /careful, /freeze, /guard, /unfreeze, /jstack-upgrade.
    ```
  </Accordion>
</AccordionGroup>
