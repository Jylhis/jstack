`/freeze` restricts all file edits to a single directory. When you're debugging a billing bug, you don't want Claude accidentally "fixing" unrelated code in `src/auth/`.

## Example

```
/freeze src/billing

Claude: Edits restricted to src/billing/. Run /unfreeze to remove.

        [Later, Claude tries to edit src/auth/middleware.ts]

Claude: BLOCKED — Edit outside freeze boundary (src/billing/).
        Skipping this change.
```

## How it works

`/freeze` blocks Edit and Write tools outside the specified path. It uses Claude Code's PreToolUse hooks — session-scoped, no configuration files.

<Note>
  Bash commands like `sed` can still modify files outside the boundary — `/freeze` is accident prevention, not a security sandbox.
</Note>

## Auto-freeze

[`/investigate`](/skills/investigate) activates `/freeze` automatically — it detects the module being debugged and restricts edits to that directory.
