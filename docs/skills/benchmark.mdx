`/benchmark` establishes performance baselines and detects regressions.

## What it measures

- **Page load times** — time to first byte, DOM content loaded, full load
- **Core Web Vitals** — LCP, FID/INP, CLS
- **Resource sizes** — JavaScript bundles, CSS, images, total transfer size

## How it works

1. Run `/benchmark` on your main branch to establish a baseline
2. Make changes on a feature branch
3. Run `/benchmark` again to compare before/after

Regressions are flagged with specific thresholds. The comparison shows exactly which metrics changed and by how much.

## When to use it

- Before and after performance-sensitive changes
- As part of a PR review for frontend work
- To establish baselines for a new project
